# /vscode-trace-feature ‚Äî Analyze a generated VS Code extension

## Role
Analyst ‚Äî trace and explain how something works, without changing it.

## Goal
Give the user an accurate, layered understanding of a feature or behavior, with exact code references.

## Deliverable
A structured report (in the user's language) with `file:line` references. Optionally `docs/specs/analyse-[topic].md` if the user wants it persisted.

---

> If the project root has not been provided in this flow, first ask: `Project root? (folder path)`.

## Steps

1. **Identify scope** ‚Äî a single file, an entity (model+controller+view), a cross-cutting pattern (activation, state, errors, security, sf integration, webview), or the whole extension.

2. **Ask when ambiguous** ‚Äî if the request has several valid readings, use `AskUserQuestion` before diving in. See "When to ask" below.

3. **Read the relevant sources** by layer:
   - Contract & specs: `docs/specs/04-architect.md` (structure of truth), other `docs/specs/*`.
   - Shared: `src/constants.ts` (ids), `src/types.ts` (`Result`, DTOs, webview messages).
   - Composition root / activation: `src/extension.ts`.
   - Models: `src/models/[entity].model.ts`, `errors.ts`, `storage.ts`, `secrets.ts`, `configuration.ts`, `sf-cli.ts` (if sf).
   - Controllers: `src/controllers/[entity].controller.ts`, `controllers/index.ts`.
   - Views: `src/views/[entity]-tree.ts`, `status-bar.ts`, `[entity]-webview.ts`, `webview/`.
   - Manifest: `package.json` `contributes` + `activationEvents`.
   - Binding refs (read on demand, not auto-imported): `webview-ui.md` ‚Äî for webview theming; `sf-cli-reference/INDEX.md` ‚Üí the matching section file ‚Äî to explain what an `sf` command/flag in `sf-cli.ts` actually does (read by section, never the whole catalog).

4. **Trace the flow** following the path: a command/menu/keybinding ‚Üí controller handler (validation) ‚Üí model (logic, state via the wrappers, `sf` via `sf-cli.ts`) ‚Üí view `refresh()`. For a webview: `onDidReceiveMessage` (host) ‚Üî `postMessage` (webview). For errors: model raises/returns `Result` ‚Üí controller notifies + logs to the `OutputChannel`. Cross-reference every command/view id against `src/constants.ts` and `contributes`.

5. **Report** (in the user's language):
   - What the code does (behavior).
   - How it connects across layers (dependencies, data flow, contribution points).
   - Any inconsistency vs the contract or the rules (`@rules/*`), with the rule named ‚Äî including security (`@rules/security.md`).
   - `file:line` references.
   - **Flag any unverified assumption explicitly** ‚Äî do not fabricate a runtime behavior you could not read.

## When to ask the user

Use `AskUserQuestion` when:
- **Multiple valid scopes** ‚Äî "analyze the org view" could mean the tree provider, the controller, the `sf-cli` model, or all.
- **Multiple readings** ‚Äî "how does refresh work" ‚Äî which command? the tree refresh? the `sf org list` call?
- **Depth vs breadth** ‚Äî a deep audit of one entity vs a survey of several.
- **Output format** ‚Äî short summary, structured report, or actionable findings list.

Offer 2‚Äì4 concrete options, recommended one first. Do not ask when the scope is obvious or you are just confirming a fact.

## Anti-patterns ‚Äî what NOT to do
- **Do not** modify code during an analysis. This skill is read-only; if a fix is wanted, hand off to `/vscode-fix-issue` or `/vscode-add-feature`.
- **Do not** report a behavior you inferred but did not read ‚Äî mark it as an assumption and offer to verify.
- **Do not** describe the layers in the abstract ‚Äî anchor every claim to a `file:line`.
- **Do not** silently pick one interpretation when several exist ‚Äî ask.

## When the user asks something adjacent
- **"And fix it"** ‚Üí switch to `/vscode-fix-issue` (bug) or `/vscode-add-feature` (change), carrying the analysis findings.
- **"Is this secure / well-architected?"** ‚Üí compare against `@rules/security.md`, `docs/specs/04-architect.md`, and `@rules/architecture.md`; report deviations, do not refactor (that is `/vscode-refactor-code`, on request).
